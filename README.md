# Skillbox diagnostic MVP

Fullscreen web2app-flow диагностики состояния и карьерного перехода.

Внутри: интерактивная диагностика, rule-based скоринг, архетип изменений, карта перехода, подбор профессиональных сред, стартовый бонус и финальный CTA.

## Запуск

```bash
node server.mjs
```

Локальный адрес: `http://localhost:4173`

Если в окружении есть `npm`, можно также использовать `npm start`.

## Публикация на GitHub Pages

Проект можно опубликовать как статический сайт через GitHub Pages:

1. Залить файлы репозитория на GitHub.
2. В настройках репозитория открыть `Settings → Pages`.
3. В `Build and deployment` выбрать `Deploy from a branch`.
4. Выбрать ветку `main` и папку `/root`.

Важно: GitHub Pages не запускает `server.mjs`, поэтому `POST /save-step` не будет писать ответы в `data/sessions.jsonl`. Сам квиз будет работать, а события сохранятся в `localStorage`/очереди браузера. Для реального сбора лидов нужен серверный хостинг вроде Vercel/Render или отдельный endpoint.

## Данные

После каждого шага фронтенд отправляет `POST /save-step`. Локальный MVP-сервер пишет события в:

```text
data/sessions.jsonl
```

Если endpoint временно недоступен, события остаются в `localStorage` и отправляются повторно при следующей возможности.

## Конфиги

Основные точки изменения находятся в `app.js`:

- `screensConfig` — порядок экранов, тексты, CTA и типы экранов.
- `questionsConfig` — варианты выбора.
- `rules` — правила скоринга.
- `archetypes` — архетипы финального результата.
- `directionRecommendations` — рекомендации профессиональных сред.
- `offers` — стартовые бонусы.
- `finalCtas` — финальные CTA.
